# Documentation Reports

**Purpose**: Automated documentation lifecycle and accuracy validation reports

**Status**: ✅ Active - Reports generated automatically

---

## Reports Generated

All reports are generated by automated scripts and CI/CD workflows:

### Weekly Reports

- **`stale-docs-report.md`** - Stale documentation detection
  - Generated by: `pnpm docs:check:stale`
  - Frequency: Weekly

- **`assessments-report.md`** - Assessment file management
  - Generated by: `pnpm docs:manage:assessments`
  - Frequency: Weekly

- **`broken-references.md`** - Reference validation
  - Generated by: `pnpm docs:check:references`
  - Frequency: Weekly

- **`accuracy-report.md`** - Documentation accuracy validation
  - Generated by: `pnpm docs:validate:accuracy`
  - Frequency: Weekly

### Monthly Reports

- **`duplicates-report.md`** - Duplicate documentation detection
  - Generated by: `pnpm docs:check:duplicates`
  - Frequency: Monthly

### Quarterly Reports

- **`archive-review.md`** - Archive review and deletion recommendations
  - Generated by: `pnpm docs:review:archive`
  - Frequency: Quarterly

---

## CI/CD Integration

**Workflow**: `.github/workflows/docs-lifecycle.yml`

**Schedule**:
- **Weekly**: Every Monday at 9 AM UTC
- **Monthly**: First Monday of month
- **Quarterly**: First Monday of quarter

**Reports**: Uploaded as GitHub Actions artifacts

---

## Manual Generation

Generate reports manually:

```bash
# Weekly reports
pnpm docs:lifecycle:weekly

# Monthly reports (includes duplicates)
pnpm docs:lifecycle:monthly

# Quarterly reports (includes archive review)
pnpm docs:lifecycle:quarterly

# Individual reports
pnpm docs:check:stale
pnpm docs:manage:assessments
pnpm docs:check:references
pnpm docs:validate:accuracy
pnpm docs:check:duplicates
pnpm docs:review:archive
```

---

## Report Format

All reports follow consistent format:

```markdown
# Report Name

**Generated**: ISO timestamp
**Summary**: Statistics and counts

## Issues (if any)
- Errors (🔴)
- Warnings (🟡)
- Info (🔵)

## Recommendations
- Action items
- Suggested fixes
```

---

## Using Reports

1. **Review errors first** - Fix critical issues immediately
2. **Address warnings** - Fix within sprint/cycle
3. **Review recommendations** - Action items for improvement
4. **Archive old reports** - Clean up quarterly
