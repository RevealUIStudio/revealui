## Current Architecture

### API Routes Structure

**Memory Routes** (`apps/cms/src/app/api/memory/`):
- `GET /api/memory/episodic/:userId` - Get memories
- `POST /api/memory/episodic/:userId` - Create memory
- `PUT /api/memory/episodic/:userId/:memoryId` - Update memory
- `DELETE /api/memory/episodic/:userId/:memoryId` - Delete memory

**Context Routes** (`apps/cms/src/app/api/memory/context/`):
- `GET /api/memory/context/:sessionId/:agentId` - Get context
- `POST /api/memory/context/:sessionId/:agentId` - Update context
- `DELETE /api/memory/context/:sessionId/:agentId` - Remove context key

**Conversations Routes** (Auto-generated by RevealUI CMS):
- `GET /api/conversations` - List conversations
- `GET /api/conversations/:id` - Get conversation
- `POST /api/conversations` - Create conversation
- `PATCH /api/conversations/:id` - Update conversation
- `DELETE /api/conversations/:id` - Delete conversation

### Database Schema

**Tables** (`packages/db/src/core/agents.ts`):
- `agent_contexts` - Agent context storage
- `agent_memories` - Long-term memory
- `conversations` - User-agent conversations
- `agent_actions` - Action audit log

**Key Fields**:
- All tables have `id` (text primary key)
- All tables have `version` (integer, for schema versioning)
- Timestamps: `created_at`, `updated_at`
- Vector embeddings for semantic search (pgvector)

### Current Sync Implementation

**Hooks** (`packages/sync/src/hooks/`):
- `useAgentContext` - Uses `useShape` from `@electric-sql/react`
- `useAgentMemory` - Uses `useShape` from `@electric-sql/react`
- `useConversations` - Uses `useShape` from `@electric-sql/react`

**Pattern**:
```typescript
// Reads: Direct ElectricSQL shape subscription
const { data } = useShape({
  url: shapeUrl,
  params: {
    table: 'agent_contexts',
    where: 'agent_id = $1',
    params: { '1': agentId },
  },
})

// Writes: RevealUI API (not ElectricSQL REST)
await updateAgentContext(sessionId, agentId, updates, authToken)
```

**API Client** (`packages/sync/src/utils/revealui-api.ts`):
- Calls RevealUI CMS API endpoints
- Handles authentication tokens
- Maps agentId to userId for memory routes

## Key Findings

### ✅ What's Working

1. **ElectricSQL Integration**
   - `@electric-sql/react` hooks working
   - Shape subscriptions functional
   - Real-time sync working

2. **Hybrid Approach**
   - Reads via ElectricSQL shapes ✅
   - Writes via RevealUI API ✅
   - No unverified ElectricSQL REST mutations ✅

3. **Database Schema**
   - Drizzle ORM tables defined
   - Type-safe with TypeScript
   - Vector embeddings supported

4. **API Routes**
   - Memory routes exist and working
   - Context routes exist and working
   - Conversations auto-generated by CMS

### ⚠️ What Needs Work

1. **No Shape Proxy Routes**
   - ElectricSQL shapes connect directly to service
   - No authentication/authorization layer
   - No row-level filtering at proxy level
   - **Impact**: Security concern, users could access all data

2. **No Transaction IDs**
   - Mutations don't return transaction IDs
   - Can't match optimistic writes with sync updates
   - **Impact**: No optimistic mutations possible

3. **No Authentication in API Routes**
   - Routes don't validate sessions
   - No user context in routes
   - **Impact**: Security concern

4. **Direct ElectricSQL Connection**
   - Client connects directly to ElectricSQL service
   - No proxy layer for authentication
   - **Impact**: Security and control concerns
