# Changesets

Hello and welcome! This folder has been automatically generated by `@changesets/cli`, a tool to help manage versioning and changelogs for packages in this monorepo.

## What are Changesets?

Changesets are a way to document changes you've made to a package. They help us:
- Track what changed in each release
- Automatically update version numbers
- Generate changelogs
- Coordinate releases across multiple packages

## Adding a Changeset

When you make changes that should be published, you need to add a changeset. To create a changeset, run:

```bash
pnpm changeset
```

This will:
1. Ask which packages you've changed
2. Ask what kind of change it is (major, minor, or patch)
3. Ask you to write a summary of the change
4. Create a new file in `.changeset/` with your changeset

## Types of Changes

- **Major**: Breaking changes that require users to update their code
- **Minor**: New features that are backward compatible
- **Patch**: Bug fixes and small changes that are backward compatible

## Example Changeset

```markdown
---
"reveal": minor
---

Add new `useTheme` hook for theme management
```

## Release Process

1. Create changesets for your changes
2. When ready to release, run `pnpm changeset version` to update versions
3. Review the generated changelog
4. Commit the changes
5. Run `pnpm publish:stable` or `pnpm publish:beta` to publish
