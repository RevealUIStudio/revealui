# @revealui/generated

Generated code for the RevealUI framework. This package contains all auto-generated code from various sources across the monorepo.

## ⚠️ Important

**DO NOT MODIFY FILES IN THIS PACKAGE MANUALLY**

All files in this package are automatically generated by scripts and will be overwritten. Instead, modify the source files and regenerate.

## Package Structure

This package is organized by category to support different types of generated code:

```
src/
├── types/          # Generated TypeScript types
│   ├── cms.ts     # CMS configuration types
│   └── supabase.ts # Supabase database types
├── functions/      # Generated utility functions (future)
├── components/    # Generated React components (future)
├── hooks/         # Generated React hooks (future)
├── prompts/       # Generated AI prompts (future)
├── plans/         # Generated execution plans (future)
├── agents/        # Generated AI agents (future)
└── tools/         # Generated utility tools (future)
```

## Usage

### Importing Types

```typescript
// Import all generated types
import type { Config, Page, Post } from '@revealui/generated/types'

// Import specific type categories
import type { Config } from '@revealui/generated/types/cms'
import type { Database } from '@revealui/generated/types/supabase'

// Import from main entry (re-exports all categories)
import type { Config } from '@revealui/generated'
```

### Importing Other Generated Code

As the codebase matures, you'll be able to import:

```typescript
// Functions
import { someGeneratedFunction } from '@revealui/generated/functions'

// Components
import { SomeGeneratedComponent } from '@revealui/generated/components'
```

## Generation Scripts

### CMS Types

CMS types are generated from `apps/cms/revealui.config.ts`:

```bash
pnpm generate:revealui-types
```

This runs:
1. `pnpm --filter=cms generate:revealui-types` - Generates types in CMS app
2. `tsx scripts/types/copy-generated-types.ts --cms` - Copies to this package

### Supabase Types

Supabase types are generated from the Supabase database schema:

```bash
pnpm generate:supabase-types
```

This runs:
1. `supabase gen types` - Generates types from Supabase
2. `tsx scripts/types/copy-generated-types.ts --supabase` - Copies to this package

### Copy All Types

To copy all generated types:

```bash
pnpm copy:generated-types
```

## Development

### Type Checking

```bash
pnpm --filter @revealui/generated typecheck
```

### Adding New Generated Code

When adding new categories of generated code:

1. Create a new directory in `src/` (e.g., `src/new-category/`)
2. Create an `index.ts` file with at least `export {}` (for TypeScript module validation)
3. Add exports to `package.json`:
   ```json
   "./new-category": {
     "types": "./src/new-category/index.ts",
     "import": "./src/new-category/index.ts"
   }
   ```
4. Re-export from main `src/index.ts`:
   ```typescript
   export * from './new-category'
   ```
